The wind_forecast/ pipeline is in place under your workspace. Here is what was built and how to run it.

Project layout
wind_forecast/
├── data/                    ← place hourly.csv here (+ loader, features, splitter)
├── main.py                  ← runs all 4 seasons (or subset via CLI)
├── config.py
├── models/                  ← forecasters, tuning, stacking, saved/{season}/
├── evaluation/              ← backtesting, metrics, probabilistic
├── explainability/          ← SHAP
├── plots/                   ← 10 PNGs per season
└── results/                 ← metrics, plots, production forecasts
Note: Your spec lists data/ twice (CSV vs Python modules). Both live under data/: hourly.csv next to loader.py, features.py, splitter.py, with data/__init__.py as the package.

Before running
Put your dataset at wind_forecast/data/hourly.csv (folder is empty except .gitkeep).
Install dependencies:
pip install -r wind_forecast/requirements.txt
Expected target/exog columns: WS, WD, AT, RH, ET, SR, VP, DPT, PCP, ST, plus a datetime column (or CIMIS-style Date + Hour (PST) — auto-mapped in loader.py).
Run
cd wind_forecast
python main.py              # all seasons, 20 Bayesian trials per tunable model
python main.py --fast       # 2 trials (smoke test)
python main.py --season spring summer   # subset of seasons
python main.py --strict     # stop on first error (default: log and continue)

Overnight / unattended:
- By default, errors are appended to results/pipeline_errors.log and the run
  continues with the next model step or season.
- Use --strict only if you want the old fail-fast behaviour.
Spring was validated end-to-end in fast mode (tuning → validation → test → probabilistic → SHAP → plots → save → production).
